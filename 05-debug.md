# Debug

Um nível de log usado para eventos considerados úteis durante a depuração de software quando informações mais granulares são necessárias.

É mais difícil definir quais informações logar no nível DEBUG do que defini-las para os outros níveis.

Resumindo, queremos registrar qualquer informação que nos ajude a identificar o que deu errado no nível DEBUG.

Exemplos concretos para usar o nível DEBUG são:

- Mensagens de erro quando uma solicitação HTTP recebida foi malformada, resultando em um status HTTP 4xx valores variáveis ​​na lógica de negócios.
- Quando você está depurando um problema ou um erro na aplicação.
- Quando você deseja entender melhor como a aplicação está funcionando internamente.
- Quando você deseja rastrear o fluxo de execução de uma aplicação para identificar problemas ou otimizar o desempenho.
- Quando você deseja fornecer informações detalhadas sobre as operações realizadas pelo código da aplicação.

O nível DEBUG pode ser usado de forma mais generosa do que os níveis acima.
